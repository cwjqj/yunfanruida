# 雷达人数统计核心模块说明

更新时间：2026-07-10

当前现场验证只以 `scripts/lua/radar_iot_task.lua` 为主任务脚本。该文件已同步为 `scripts/lua/radar_people_count_core.lua` 的核心人数统计版本，目标是先保证 `people_count` 准确上云，姿态、床区展示、复杂目标对象等功能暂不作为主判断依据。

## 1. 原始帧解析依据

雷达 UART 帧格式：

```text
53 59 | Control | Command | Len_H Len_L | Data... | Sum | 54 43
```

- 长度字段为大端格式。
- 校验为从 `53 59` 到数据区最后一字节求和后取低 8 位。
- `0x82 / 0x02` 轨迹目标按 11 字节解析：`id,size,feature,x(2),y(2),height(2),speed(2)`。
- `x/y/speed` 使用雷达最高位符号格式，例如 `80 08` 表示 `-8cm`，不是普通 int16 补码。

## 2. 人数统计口径

`people_count` 只来自脚本内部确认后的轨迹，不再直接采用雷达上报的目标数量或人数统计值。

- `target_count`：只表示当前轨迹帧原始目标数，用于排查，不参与人数统计。
- `realtime_people_count`：只保存 `0x86 / 0x0A` 原始实时人数，用于对照，不参与人数统计。
- `accurate_people_count`：只保存 `0x86 / 0x0C` 原始精准人数，用于对照，不参与人数统计。
- `people_count`：只统计 confirmed track，确认条件是连续命中、累计位移和运动证据均满足阈值。
- 不设置“床下最多 1 人”这类业务上限，现场真实多人时允许统计多人；昨天“最多 1 人”只作为校验真值，不写成固定限制。

## 3. 轨迹保留与减人

确认后的轨迹进入 `active`，丢帧后进入 `holding`，继续用最后一次宿舍区域内坐标上报主目标坐标。

- 最后坐标必须在宿舍固定范围内才会更新：`X=[-165,165]cm`，`Y=[0,500]cm`。
- 进入门口区域后丢失超过 `EXIT_GRACE_SEC`，认为离开并减人。
- 下方近端区域内“运动后丢失”超过 `LOWER_MOVING_LOST_EXIT_SEC`，认为离开并减人。
- `human_present=0`、实时人数为 0、精准人数不阻塞时，持续 `FALLBACK_ZERO_SECONDS` 后清零。
- 雷达串口长时间无有效帧超过 `RADAR_STALE_SECONDS`，主动清零并上报，避免断雷达后云端旧人数残留。

## 4. 上云字段

必须保留以下字段用于现场定位问题：

| 字段 | 用途 |
|---|---|
| `people_count` | 最终人数，以此判断现场人数 |
| `target_count` | 雷达当前轨迹帧原始目标数，仅排查 |
| `realtime_people_count` | 雷达 `0x86/0x0A` 原始实时人数，仅排查 |
| `accurate_people_count` | 雷达 `0x86/0x0C` 原始精准人数，仅排查 |
| `radar_frame_count` | 有效雷达帧计数 |
| `radar_ready` | 是否收到过有效雷达帧 |
| `uart_bytes` | 最新一次 UART 接收原始 HEX 文本 |

`uart_bytes` 是最新一次 `UartGetRecChAndDel(1)` 读取到的原始字节 HEX 字符串，例如：

```text
53 59 82 02 00 0B 00 01 02 80 08 00 2E 00 B8 00 06 B2 54 43
```

这个字段用于判断问题来源：如果 `uart_bytes` 中原始帧已经持续多目标，则优先查雷达安装、范围、干扰边；如果原始帧正常但 `people_count` 异常，则查 Lua 轨迹确认和清退逻辑。

## 5. 现场验证顺序

1. DTU 启动后确认 `radar_ready=1`、`radar_frame_count` 持续增加、`uart_bytes` 有 HEX 数据。
2. 空场确认 `people_count=0`，同时记录 `target_count/realtime_people_count/accurate_people_count` 是否误报。
3. 1 人进入床下或下方区域，观察完整移动轨迹后 `people_count` 是否变为 1。
4. 人静止后如果轨迹丢帧，`people_count` 应保持 1，主坐标保持最后一次宿舍区域内坐标。
5. 人从门口或下方区域离开，确认超过对应清退时间后 `people_count` 下降。
6. 拔掉雷达串口或雷达断电但 DTU 仍在线，确认超过 `RADAR_STALE_SECONDS` 后 `people_count=0` 并上报。

## 6. 当前脚本结论

主任务脚本：`scripts/lua/radar_iot_task.lua`

参考核心脚本：`scripts/lua/radar_people_count_core.lua`

两者当前内容保持一致。后续只要现场验证人数统计，优先修改核心脚本并同步到主任务脚本，不能再把 `target_count`、`0x86/0x0A` 或 `0x86/0x0C` 直接写回 `people_count`。
