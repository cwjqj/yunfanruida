# 云帆瑞达毫米波雷达项目手册

> 本文件只负责“从哪里开始看”。工程的协议、参数查询、运行环境、云端 Lua 框架和云平台数据点，统一以 [`通用/文档/云帆瑞达毫米波雷达通用工程文档.md`](通用/文档/云帆瑞达毫米波雷达通用工程文档.md) 为核心。
>
> 现有目录结构保持不变：`通用`、`侧装`、`顶装` 三个目录继续按安装方式和用途保存文件；本文和通用总文档只是把阅读主线理顺，不移动脚本、不删除历史记录。

---

## 1. 项目一句话

将云帆瑞达 R60BMP1 60G 毫米波雷达接入 Air780EP 4G DTU，解析人体存在、运动、体动、轨迹和人数，通过 MQTT 物模型上报到银尔达 / YED IoT 云平台，服务于寝室“上床下桌”等安装场景。

---

## 2. 唯一推荐阅读顺序

### 第一次接手工程

1. [`通用/文档/云帆瑞达毫米波雷达通用工程文档.md`](通用/文档/云帆瑞达毫米波雷达通用工程文档.md)：先建立完整主线。
2. [`通用/文档/雷达模块内部参数汇总.md`](通用/文档/雷达模块内部参数汇总.md)：看当前设备读回参数和完整查询帧。
3. [`侧装/文档/当前/项目梳理与交接手册.md`](侧装/文档/当前/项目梳理与交接手册.md)：看当前侧装现场状态、风险和交接清单。
4. [`侧装/脚本/lua/radar_iot_task.lua`](侧装/脚本/lua/radar_iot_task.lua)：看当前 DTU 部署代码。
5. 需要深入协议时，再读 [`通用/文档/串口通信协议详解.md`](通用/文档/串口通信协议详解.md)。

### 现场调参数

1. 先读通用总文档第 2、4、8 节。
2. 使用 [`通用/脚本/python/query_radar_settings.py`](通用/脚本/python/query_radar_settings.py) 保存当前读回。
3. 根据场景使用 `通用/脚本/python/set_*.py` 或对应目录的一键配置脚本。
4. 设置后重新查询，不能只看“发送成功”。

### 排查“雷达没数据”

按顺序使用：

```text
listen_radar.py → radar_parser.py → query_radar_settings.py → DTU Lua 日志 → 云平台数据点
```

## 4. 当前基线

以下信息用于快速判断工程当前状态，详细内容见通用总文档：

| 项目 | 当前基线 |
|---|---|
| 雷达 | R60BMP1 60G |
| PC 串口 | `COM3`，115200，8N1 |
| DTU 串口 | UART1，115200，8N1 |
| 云通道 | `nid=1` |
| 当前实物安装记录 | 侧装/倾斜，X=0°、Y=0°、Z=30°，高度 220cm |
| 当前 DTU 主脚本 | `侧装/脚本/lua/radar_iot_task.lua` |
| 顶装人数 DTU 脚本 | 当前入口 `顶装/脚本/lua/radar_iot_task.lua`；版本副本 `versions/count/radar_iot_task_v1.3.lua`；`pver=Radar_Top_Count_1.3` |
| 顶装轨迹实验脚本 | 独立入口 `顶装/脚本/lua/radar_iot_task_track.lua`；版本副本 `versions/track/radar_iot_task_track_v2.0.lua`；`pver=Radar_Top_Track_2.0` |
| 顶装姿态实验脚本 | 独立入口 `顶装/脚本/lua/radar_iot_task_posture.lua`；当前版本副本 `versions/posture/radar_iot_task_posture_v3.1.lua`；`pver=Radar_Top_Posture_3.1`；保留 `v3.0` 回退版 |
| 通信帧 | `53 59 ... 54 43`，长度大端，校验和低 8 位 |
| 轨迹目标 | 每目标 11 字节，坐标/速度使用符号位格式 |
| 业务人数口径 | 当前核心版以 confirmed tracks 为准，雷达实时/精准人数用于参考和校准 |
| 顶装人数口径 | 软件范围内 confirmed tracks 为最终人数；实时/精准人数和门事件只作全波束诊断 |

> 顶装记录已经有单独的实物验证，但不应直接覆盖侧装 Lua 的场景常量；切换安装方式时必须重新标定坐标、区域和姿态阈值。

## 6. 文档维护规则

1. 通用内容先改通用总文档，再在侧装/顶装文档补充场景差异。
2. 云平台数据点、Lua 字段、协议命令发生变化时，必须同步更新通用总文档。
3. “当前参数”必须标注读取时间和设备来源；旧记录放在原有归档或备份目录。
4. 不移动现有工程文件，不用实验版覆盖当前主脚本。
5. 新人只需要先读本文和通用总文档，就能知道下一步该看哪个文件。
6. 顶装 Lua 每次更新必须同时保留带版本号的副本 `radar_iot_task_v<版本号>.lua`，回复、脚本 `pver` 和测试记录使用同一版本号。
