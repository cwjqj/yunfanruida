# 云帆瑞达 60G 毫米波雷达串口通信协议详解

## 一、基本串口参数

| 参数 | 值 |
|------|-----|
| 接口电平 | TTL (3.3V) |
| 波特率 | 115200 bps |
| 停止位 | 1 |
| 数据位 | 8 |
| 奇偶校验 | 无 |

---

## 二、帧结构定义

### 通用帧格式

```
0x53 0x59 | Control | Command | Len_H Len_L | Data... | Sum | 0x54 0x43
  2 Byte     1 Byte    1 Byte      2 Byte      n Byte  1 Byte   2 Byte
```

### 字段说明

| 字段 | 长度 | 说明 |
|------|------|------|
| **帧头** | 2 Byte | 固定 `0x53, 0x59` |
| **控制字** | 1 Byte | 标识数据包类型 |
| **命令字** | 1 Byte | 对当前数据内容进行标识 |
| **长度标识** | 2 Byte | 等于数据的具体字节长度（大端序） |
| **数据** | n Byte | 根据实际功能定义 |
| **校验码** | 1 Byte | 帧头+控制字+命令字+长度标识+数据 求和后取低八位 |
| **帧尾** | 2 Byte | 固定 `0x54, 0x43` |

### 校验码计算示例

存在信息查询指令：`53 59 80 81 00 01 0F [sum] 54 43`

sum = (0x53 + 0x59 + 0x80 + 0x81 + 0x00 + 0x01 + 0x0F) = 0x01BD → 取低字节 = **0xBD**

完整指令：`5359808100010FBD5443`

## 四、各功能命令详解

### 4.1 系统功能

#### 4.1.1 心跳包（主动上报，1 分钟一次）

```
53 59 01 01 00 01 0F [sum] 54 43
```

| 字节 | 值 | 含义 |
|------|-----|------|
| 0-1 | `53 59` | 帧头 |
| 2 | `01` | 控制字：心跳包标识 |
| 3 | `01` | 命令字 |
| 4-5 | `00 01` | 数据长度：1 字节 |
| 6 | `0F` | 数据内容 |
| 7 | `[sum]` | 校验码 |
| 8-9 | `54 43` | 帧尾 |

#### 4.1.2 固件版本查询

**下发：**
```
53 59 02 A4 00 01 0F [sum] 54 43
```

**回复：**
```
53 59 02 A4 [len] [len 字节固件版本] [sum] 54 43
```

#### 4.1.3 初始化完成状态

**查询下发：**
```
53 59 05 81 00 01 0F [sum] 54 43
```

**回复：**
```
53 59 05 81 00 01 [01/00] [sum] 54 43
```

| 数据值 | 含义 |
|--------|------|
| `01` | 初始化已完成 |
| `00` | 初始化未完成 |

#### 4.1.4 初始化完成信息（主动上报）

```
53 59 05 01 00 01 0F [sum] 54 43
```

### 4.2 雷达安装信息（控制字 `0x06`）

#### 4.2.1 安装角度设置/查询

**下发：**
```
53 59 06 01 00 06 [X 轴角度 2B] [Y 轴角度 2B] [Z 轴角度 2B] [sum] 54 43
```

| 字节位置 | 长度 | 说明 |
|----------|------|------|
| 0-1 | - | 帧头 `53 59` |
| 2 | 1 | 控制字 `06` |
| 3 | 1 | 命令字 `01` |
| 4-5 | 2 | 数据长度 `00 06` |
| 6-7 | 2 | X 轴角度 |
| 8-9 | 2 | Y 轴角度 |
| 10-11 | 2 | Z 轴角度 |
| 12 | 1 | 校验码 |
| 13-14 | 2 | 帧尾 `54 43` |

- 实际角度范围：-180° ~ 180°；协议值范围为 -18000 ~ 18000
- **实际角度扩大 100 倍后传输**
- Z 轴表示下倾角度，默认 30°
- 支持 30°（侧装）、90°（顶装）

顶装 90°时，Z 轴协议值为 `90 × 100 = 9000 = 0x2328`，X/Y 轴均为 0。完整设置帧为：

```text
53 59 06 01 00 06 00 00 00 00 23 28 04 54 43
```

其中校验码 `04` 为从帧头 `53 59` 到数据 `23 28` 的逐字节累加低 8 位。`03 84` 等于十进制 900，只代表 9.00°，不能用于顶装 90°。

#### 4.2.2 安装高度设置/查询

**下发：**
```
53 59 06 02 00 02 [高度信息 2B] [sum] 54 43
```

- 单位：cm，步长 1cm

#### 4.2.3 陀螺仪信息（主动上报）

```
53 59 06 04 00 08 [水平角 2B] [俯仰角 2B] [水平标准差 2B] [俯仰标准差 2B] [sum] 54 43
```

| 字节位置 | 长度 | 说明 |
|----------|------|------|
| 6-7 | 2 | 水平角 |
| 8-9 | 2 | 俯仰角 |
| 10-11 | 2 | 水平标准差 |
| 12-13 | 2 | 俯仰标准差 |

#### 4.2.4 陀螺仪异常（主动上报）

```
53 59 06 05 00 01 [状态] [sum] 54 43
```

| 数据值 | 含义 |
|--------|------|
| `00` | 正常 |
| `01` | 无传感器 |
| `02` | 测量角度与预设角度差异过大（超过 ±5° 会在 5min 上报） |

### 4.3 人体存在功能（控制字 `0x80`）

#### 4.3.1 人体存在功能开关（命令字 `0x00`）

| 方向 | 说明 |
|------|------|
| 下发 | `01`：开启，`00`：关闭 |
| 回复 | 同下发格式 |

#### 4.3.2 人体存在状态上报（命令字 `0x01`，主动上报）

状态变化时上报：

| 数据值 | 含义 |
|--------|------|
| `00` | 无人 |
| `01` | 有人 |

#### 4.3.3 运动信息上报（命令字 `0x02`，主动上报）

状态变化时上报：

| 数据值 | 含义 |
|--------|------|
| `00` | 无运动 |
| `01` | 静止 |
| `02` | 活跃 |

#### 4.3.4 体动参数上报（命令字 `0x03`，主动上报）

- 1 秒上报一次
- 数据：1 字节体动参数（0-100）

#### 4.3.5 查询命令

| 命令字 | 说明 | 下发指令示例 |
|--------|------|-------------|
| `0x80` | 查询人体存在开关 | `53 59 80 80 00 01 0F [sum] 54 43` |
| `0x81` | 查询存在信息 | `53 59 80 81 00 01 0F [sum] 54 43` |
| `0x82` | 查询运动信息 | `53 59 80 82 00 01 0F [sum] 54 43` |
| `0x83` | 查询体动参数 | `53 59 80 83 00 01 0F [sum] 54 43` |

### 4.4 人数统计配置（控制字 `0x07`）

#### 4.4.1 自动探测范围限制设置（命令字 `0x08`）

**下发：**
```
53 59 07 08 00 01 [01/00] [sum] 54 43
```

| 数据值 | 含义 |
|--------|------|
| `01` | 开启自动探测 |
| `00` | 关闭自动探测 |

**回复（开启）：**
```
53 59 07 08 [len] 01 [sum] 54 43
```

**回复（关闭）：**
```
53 59 07 08 [len] 01 + n*(X 位置 2B + Y 位置 2B) [sum] 54 43
```

#### 4.4.2 雷达探测范围设置（命令字 `0x09`）

**下发：**
```
53 59 07 09 00 09 [00] [x 正半轴 2B] [x 负半轴 2B] [y 正半轴 2B] [y 负半轴 2B] [sum] 54 43
```

- 单位：cm

#### 4.4.3 自动探测范围是否使用（命令字 `0x0C`）

| 数据值 | 含义 |
|--------|------|
| `01` | 使用 |
| `00` | 不使用 |

#### 4.4.4 标签设置（命令字 `0x11`）

**下发：**
```
53 59 07 11 00 06 [标签类型 1B] [范围类型 1B] [X 2B] [Y 2B] [sum] 54 43
```

**标签类型：**

| 值范围 | 含义 |
|--------|------|
| 0-4 | 门 |
| 5-9 | 沙发 |
| 10-14 | 床 |
| 15-19 | 干扰源 |
| 20 | 卫生间门 |
| 21-30 | 保留 |

**范围类型：**

| 值 | 含义 |
|----|------|
| `0` | 圆（X=半径，Y 不用） |
| `1` | 矩形（X=宽，Y=高） |

#### 4.4.5 其他标签相关命令

| 命令字 | 说明 |
|--------|------|
| `0x12` | 标签验证 |
| `0x13` | 标签清除 |
| `0x14` | 标签信息上报 |
| `0x15` | 标签信息上报 |

#### 4.4.6 配置文件设置探测范围（命令字 `0x17`）

**下发：**
```
53 59 07 17 [LEN] [门中心点 X 2B][门中心点 Y 2B][门宽 2B][门高 2B][范围类型 1B] + n*([坐标点 X 2B][坐标点 Y 2B]) [sum] 54 43
```

#### 4.4.7 干扰边设置（命令字 `0x1A`）

| 值 | 含义 |
|----|------|
| `1` | 左侧 0.5 米边缘，前方探测距离 4 米 |
| `2` | 右侧 0.5 米边缘，前方探测距离 4 米 |
| `3` | 酒店走廊，左右各 0.75 米 |
| `4` | 没有边界设置 |

> 仅适用于门上方中央倾斜 30° 安装模式

### 4.5 轨迹跟踪功能（控制字 `0x82`）

#### 4.5.1 轨迹跟踪开关（命令字 `0x00`）

**下发：**
```
53 59 82 00 00 01 [01/00] [sum] 54 43
```

| 数据值 | 含义 |
|--------|------|
| `01` | 开启 |
| `00` | 关闭 |

#### 4.5.2 轨迹信息上报（命令字 `0x02`）

```
53 59 82 02 [len] [索引 1B][目标大小 1B][目标特征 1B][X 位置 2B][Y 位置 2B][高度 2B][速度 2B][保留] ... [sum] 54 43
```

| 字段 | 长度 | 说明 |
|------|------|------|
| 索引 | 1 Byte | 目标索引 |
| 目标大小 | 1 Byte | - |
| 目标特征 | 1 Byte | - |
| X 位置 | 2 Byte | 有正负，16 位数据首位 0=正，首位 1=负 |
| Y 位置 | 2 Byte | 同上 |
| 高度 | 2 Byte | - |
| 速度 | 2 Byte | - |
| 保留 | - | - |

> 支持多个目标点

#### 4.5.3 轨迹信息查询（命令字 `0x82`）

**下发：**
```
53 59 82 82 00 01 0F [sum] 54 43
```

### 4.7 OTA 升级（控制字 `0x03`）

#### 4.7.1 OTA 升级开始（命令字 `0x01`）

**下发：**
```
[固件包大小 4B]
```

**回复：**
```
[每帧传输升级包大小 4B]
```

> 上位机按此大小分包

#### 4.7.2 升级包传输（命令字 `0x02`）

**下发：**
```
[包偏移地址 4B][数据包 len B]
```

**回复：**

| 数据值 | 含义 |
|--------|------|
| `00` | 失败 |
| `01` | 成功 |

#### 4.7.3 结束 OTA 升级（命令字 `0x03`）

**下发：**

| 数据值 | 含义 |
|--------|------|
| `01` | 完成 |
| `00` | 未完成 |

**回复：**
```
0F
```

## 五、硬件引脚定义

### R60BMP1 / R60BMP1-V（置顶安装）/ R60BMP1-S（倾斜安装）

三款硬件串口引脚定义一致：

**接口 1：**

| 引脚 | 功能 |
|------|------|
| 1 | GND |
| 2 | IO-1（有人无人） |
| 3 | IO-2（预留） |
| 4 | RX |
| 5 | TX |
| 6 | 5V |

**接口 2：**

| 引脚 | 功能 |
|------|------|
| 1 | GND |
| 2 | TX |
| 3 | RX |
| 4 | BOOT |
| 5 | 烧录使能 |
| 6 | 5V |

- 电平：3.3V TTL
