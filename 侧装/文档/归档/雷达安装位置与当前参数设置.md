# 雷达安装位置与当前参数设置

更新时间：2026-07-06

适用设备：云帆瑞达 R60BMP1 60G 毫米波雷达 + Air780EP 合宙 4G DTU  
当前脚本：`radar_iot_task.lua`

## 1. 推荐安装位置

当前使用场景是寝室上床下桌，床面高度按 1.8m 处理。目标是同时判断：

- 床下/桌前是否有人；
- 床上是否有人；
- 人是站立、坐下还是躺下；
- 每个目标的编号、坐标、高度、速度和姿态。

### 1.1 首选安装方式：顶装向下

建议把雷达安装在床区和桌区上方，尽量让雷达正下方覆盖床铺和桌前活动区域。

推荐位置：

- 优先安装在天花板，或者上铺床架顶部较稳定的位置；
- 安装点尽量靠近床铺和桌区的中心线；
- 雷达探测面垂直向下；
- 安装高度按实际离地高度记录，后续如果写入雷达安装高度，应使用实际离地高度，单位 cm；
- 雷达前方不要被金属床架、床板、柜门、风扇、厚窗帘等遮挡。

顶装优点：

- 对床上目标和桌前目标的高度判断更稳定；
- 更适合用 `traj_height` 区分站立、坐下、躺下；
- 更适合寝室这种上下两层活动区域。

### 1.2 备选安装方式：侧装倾斜

如果无法顶装，可以安装在墙面或床架侧面较高位置，朝床和桌区域倾斜。

建议：

- 安装高度建议高于桌面和人体坐姿高度，尽量在 2.0m 以上；
- 雷达朝向床铺和桌前活动区域；
- 倾斜安装需要同步设置雷达安装角度，否则坐标和高度判断会偏；
- 侧装时，床架金属管、护栏、梯子可能造成遮挡，实际效果通常不如顶装。

## 2. 当前坐标定义

脚本目前直接使用雷达上报的轨迹坐标：

| 字段 | 含义 | 单位 |
|---|---|---|
| `traj_x` / `primary_target_x` | 主目标 X 坐标 | cm |
| `traj_y` / `primary_target_y` | 主目标 Y 坐标 | cm |
| `traj_height` / `primary_target_height` | 主目标高度 | cm |
| `traj_speed` / `primary_target_speed` | 主目标速度 | cm/s |

当前姿态判断依赖 `x/y/height`。床区坐标还没有按现场最终标定，目前只是占位范围：

```lua
BED_X_MIN = -120
BED_X_MAX = 120
BED_Y_MIN = 0
BED_Y_MAX = 400
```

这表示脚本暂时认为：

- X 方向：-120cm 到 120cm；
- Y 方向：0cm 到 400cm；
- 落在这个矩形范围内的目标，按床区逻辑处理。

现场必须标定这个范围。标定方法：

1. 人站在床区左边界，记录云平台上报的 `primary_target_x`、`primary_target_y`；
2. 人站在床区右边界，记录坐标；
3. 人躺在床头、床尾位置，分别记录坐标；
4. 用这些实测值替换 `BED_X_MIN`、`BED_X_MAX`、`BED_Y_MIN`、`BED_Y_MAX`。

## 3. 当前姿态判断参数

当前脚本里的姿态编码：

| 编码 | 姿态 |
|---:|---|
| 0 | 无人/未知 |
| 1 | 站立 |
| 2 | 坐下 |
| 3 | 躺下 |

当前阈值：

| 参数 | 当前值 | 含义 |
|---|---:|---|
| `BED_HEIGHT_CM` | 180 | 床面高度，单位 cm |
| `BED_ON_TOLERANCE_CM` | 20 | 床面判断容差，床区内高度 >= 160cm 认为在床上 |
| `BED_SIT_HEIGHT_CM` | 220 | 床区内高度 >= 220cm 判定为坐下 |
| `STAND_HEIGHT_CM` | 130 | 非床区高度 >= 130cm 判定为站立 |
| `SIT_HEIGHT_CM` | 70 | 非床区高度 >= 70cm 且 <130cm 判定为坐下 |

当前逻辑：

```text
如果高度 <= 0：
  无人

如果目标在床区，并且高度 >= 160cm：
  高度 >= 220cm：坐下
  高度 < 220cm：躺下

如果目标不按床区处理：
  高度 >= 130cm：站立
  高度 >= 70cm：坐下
  其他：躺下
```

注意：`BED_SIT_HEIGHT_CM = 220` 是为了区分床上坐着和床上躺着。实际值需要用现场数据修正，例如分别采集：

- 人躺在床上的 `height`；
- 人坐在床上的 `height`；
- 人站在床下的 `height`；
- 人坐在桌前的 `height`。

## 4. 当前串口和雷达解析参数

| 项目 | 当前设置 |
|---|---|
| DTU 串口 | UART1 |
| 波特率 | 115200 |
| 数据位 | 8 |
| 校验 | 无 |
| 停止位 | 1 |
| 串口协议 | 任务脚本手动读取 |
| 网络通道 | `nid = 1` |
| 帧头 | `53 59` |
| 帧尾 | `54 43` |
| 校验 | 帧头到数据区求和取低 8 位 |

当前脚本会停止平台自带串口协议解析：

```lua
UartStopProRecCh(1)
PronetStopProRecCh(nid)
```

雷达轨迹帧当前按每个目标 11 字节解析：

```text
id(1) + size(1) + feature(1) + x(2) + y(2) + height(2) + speed(2)
```

当前解析的控制字：

| 控制字 | 用途 |
|---|---|
| `0x80` | 人体存在、运动状态、体动参数 |
| `0x82` | 轨迹目标 |
| `0x86` | 人数统计 |

## 5. 当前上云参数

基础版本参数：

| 字段 | 当前值 |
|---|---|
| `pver` | `Radar_DTU_1.0` |
| `fver` | `FW_1.0.0` |
| `pele` | 100 |
| `rssi` | 根据 `mobile.csq()` 换算为百分比 |

上报规则：

| 规则 | 当前值 |
|---|---|
| 开机注册 | 延时 2 秒后发送 `dreg` |
| 主循环周期 | 100ms |
| 周期心跳上报 | 180 秒 |
| 雷达未就绪保护 | 雷达就绪后才主动 `dup`，或开机超过 30 秒后允许上报 |
| 关键状态变化 | 立即触发上报 |
| 动态数据变化限频 | 最短 10 秒上报一次 |

关键状态变化包括：

- `human_present`
- `motion`
- `people_min`
- `people_max`
- `target_count`
- `posture_code`

动态变化触发阈值：

| 字段 | 变化阈值 |
|---|---:|
| `body_move` | >= 10 |
| `traj_x` | >= 20cm |
| `traj_y` | >= 20cm |
| `traj_height` | >= 20cm |
| `traj_speed` | >= 10cm/s |

## 6. 当前建议在云平台保留的数据点

### 6.1 简单数据点

| 显示名称 | 标识 ID | 数据类型 | 读写类型 | 建议范围/说明 |
|---|---|---|---|---|
| 是否有人 | `has_person` | 整数/枚举 | 只读 | 0=无人，1=有人 |
| 当前人数 | `people_count` | 整数 | 只读 | 0-10 |
| 运动状态 | `motion` | 整数/枚举 | 只读 | 0=无运动，1=静止，2=活跃 |
| 体动参数 | `body_move` | 整数 | 只读 | 0-100 |
| 主目标姿态编码 | `posture_code` | 整数/枚举 | 只读 | 0=无人/未知，1=站立，2=坐下，3=躺下 |
| 主目标姿态 | `main_posture_text` | 字符串 | 只读 | 无人、站立、坐下、躺下 |
| 目标数量 | `target_count` | 整数 | 只读 | 当前轨迹目标数量 |
| 目标编号列表 | `target_ids` | 字符串 | 只读 | 例如 `1,2,3` |
| 主目标编号 | `primary_target_id` | 整数 | 只读 | -1 表示无目标 |
| 主目标 X 坐标 | `primary_target_x` | 整数 | 只读 | cm，建议 -1000 到 1000 |
| 主目标 Y 坐标 | `primary_target_y` | 整数 | 只读 | cm，建议 -1000 到 1000 |
| 主目标高度 | `primary_target_height` | 整数 | 只读 | cm，建议 0 到 300 |
| 主目标速度 | `primary_target_speed` | 整数 | 只读 | cm/s，建议 -500 到 500 |
| 主目标姿态编码 | `primary_target_posture` | 整数/枚举 | 只读 | 0=无人/未知，1=站立，2=坐下，3=躺下 |
| 雷达就绪 | `radar_ready` | 整数/枚举 | 只读 | 0=未收到有效帧，1=已收到有效帧 |
| 雷达帧计数 | `radar_frame_count` | 整数 | 只读 | 调试用 |
| 串口字节数 | `uart_bytes` | 整数 | 只读 | 调试用 |

### 6.2 复杂数据点

复杂数据点用于按目标分别显示轨迹。当前脚本上报：

- `target0`
- `target1`
- `target2`

每个复杂数据点内部字段一致：

| 内部显示名称 | 内部标识 ID | 数据类型 | 读写类型 | 建议范围/说明 |
|---|---|---|---|---|
| 目标编号 | `id` | 整数 | 只读 | -1 到 255，-1 表示无目标 |
| X 坐标 | `x` | 整数 | 只读 | cm，建议 -1000 到 1000 |
| Y 坐标 | `y` | 整数 | 只读 | cm，建议 -1000 到 1000 |
| 目标高度 | `height` | 整数 | 只读 | cm，建议 0 到 300 |
| 目标速度 | `speed` | 整数 | 只读 | cm/s，建议 -500 到 500 |
| 姿态编码 | `posture` | 整数/枚举 | 只读 | 0=无人/未知，1=站立，2=坐下，3=躺下 |
| 姿态 | `posture_text` | 字符串 | 只读 | 无人、站立、坐下、躺下 |

复杂数据点上报格式示例：

```json
{
  "target0": {
    "id": 1,
    "x": -23,
    "y": 149,
    "height": 95,
    "speed": 0,
    "posture": 2,
    "posture_text": "坐下"
  }
}
```

## 7. 当前仍需现场确认的参数

以下参数不是最终值，需要现场采样后调整：

| 参数 | 当前值 | 需要确认的原因 |
|---|---:|---|
| `BED_X_MIN` | -120 | 需要用床左/右边界实测坐标标定 |
| `BED_X_MAX` | 120 | 需要用床左/右边界实测坐标标定 |
| `BED_Y_MIN` | 0 | 需要用床头/床尾边界实测坐标标定 |
| `BED_Y_MAX` | 400 | 需要用床头/床尾边界实测坐标标定 |
| `BED_SIT_HEIGHT_CM` | 220 | 需要用床上坐姿高度实测修正 |
| `STAND_HEIGHT_CM` | 130 | 需要用床下站姿高度实测修正 |
| `SIT_HEIGHT_CM` | 70 | 需要用桌前坐姿高度实测修正 |

建议采集四组样本后再定最终阈值：

| 场景 | 需要记录 |
|---|---|
| 床下站立 | `primary_target_x/y/height` |
| 桌前坐下 | `primary_target_x/y/height` |
| 床上躺下 | `primary_target_x/y/height` |
| 床上坐下 | `primary_target_x/y/height` |

## 8. 注意事项

1. 当前 `radar_iot_task.lua` 没有主动下发雷达安装角度、安装高度、探测范围等配置指令，只负责读取和解析雷达上报数据。
2. 如果雷达本体需要设置顶装/侧装、安装高度、探测区域，应使用厂家配置工具或串口配置指令单独写入。
3. 如果云平台返回 `dupbck rst=1`，通常是数据点未创建、类型不匹配、复杂数据点内部字段不一致，或平台不接受未定义字段。
4. 如果姿态判断不准，优先调整床区坐标和高度阈值，不要先改串口解析。
