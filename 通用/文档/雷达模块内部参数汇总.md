## 2. 串口协议

串口参数：

| 项目 | 值 |
|---|---|
| 电平 | TTL 3.3V |
| 波特率 | 115200 |
| 数据位 | 8 |
| 校验 | 无 |
| 停止位 | 1 |

通用帧格式：

```text
53 59 | Control | Command | Len_H Len_L | Data... | Sum | 54 43
```

字段说明：

| 字段 | 长度 | 说明 |
|---|---:|---|
| 帧头 | 2B | 固定 `53 59` |
| 控制字 | 1B | 数据类别 |
| 命令字 | 1B | 具体命令 |
| 长度 | 2B | 数据区长度，大端 |
| 数据 | nB | 命令数据 |
| 校验 | 1B | 帧头到数据区求和取低 8 位 |
| 帧尾 | 2B | 固定 `54 43` |

## 3. 常用控制字

| 控制字 | 含义 | 当前项目用途 |
|---|---|---|
| `0x01` | 心跳 | 可记录，不作为业务主数据 |
| `0x02` | 产品信息 | 查询固件版本 |
| `0x05` | 工作状态 | 查询初始化状态 |
| `0x06` | 安装方式 | 查询/设置安装角度、安装高度 |
| `0x07` | 人数统计配置 | 探测范围、标签、干扰边 |
| `0x80` | 人体存在 | 有人/无人、运动状态、体动 |
| `0x82` | 轨迹跟踪 | 目标坐标、高度、速度 |
| `0x86` | 人数统计 | 实时/精准人数和相关时间参数 |

## 6. 查询方法

电脑直连 USB-TTL：

1. 雷达 TX 接 USB-TTL RX。
2. 雷达 RX 接 USB-TTL TX。
3. GND 共地。
4. 供电按模块要求。
5. 修改 `query_radar_settings.py` 中的串口号。
6. 执行：

```powershell
python query_radar_settings.py
```

通过 DTU 查询时，要确保任务脚本不会同时消费串口回复；推荐临时停用任务脚本，或后续增加“服务器下发 HEX 转发到雷达 UART”的调试命令。

安装高度设置为 2.2m 的 PC 串口指令：

```powershell
python set_radar_height.py --height-cm 220
```

对应设置帧为：

```text
53 59 06 02 00 02 00 DC 92 54 43
```

本次设置后查询返回：

```text
53 59 06 82 00 02 00 DC 12 54 43
```

宿舍固定边界设置：

```powershell
python set_radar_boundary.py --x-pos-cm 165 --x-neg-cm 165 --y-pos-cm 500 --y-neg-cm 0
```

对应设置帧为：

```text
53 59 07 09 00 09 00 00 A5 00 A5 01 F4 00 00 04 54 43
```

本次设置后查询返回：

```text
53 59 07 89 00 09 00 00 A5 00 A5 01 F4 00 00 84 54 43
```

## 7. 设置方向

寝室上床下桌建议：

| 参数 | 建议 |
|---|---|
| 安装方式 | 优先顶装向下；无法顶装再使用侧装倾斜 |
| 安装角度 | 顶装需设置为对应顶装角度；当前读回是 Z=30 度 |
| 安装高度 | 写入实际离地高度，单位 cm |
| 探测范围 | 只覆盖本床位和桌前，避免覆盖邻床/走廊 |
| 床标签 | 建议后续配置床区标签 |
| 干扰源标签 | 对风扇、窗帘、晾衣区等设置干扰源 |
| 人体存在 | 保持开启 |
| 轨迹跟踪 | 保持开启 |
| 人数统计 | 保持开启 |

## 8. 与 DTU 脚本的关系

当前 `radar_iot_task.lua` 只读取和解析雷达主动上报，不主动写入雷达模块内部设置。

也就是说：

- 安装角度不是脚本设置的；
- 安装高度不是 DTU 主脚本设置的；本次已由 PC 串口脚本 `set_radar_height.py` 写入为 220cm；
- 探测范围不是脚本设置的；
- 床/门/干扰源标签不是 DTU 主脚本设置的；床铺标签可通过 `configure_dorm_bunk_radar.py --set-bed-label` 可选下发。

这些参数如果要改，需要通过厂家工具、PC 串口脚本，或后续新增 DTU 下发转发功能完成。
