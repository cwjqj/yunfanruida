### 1.1 宿舍坐标定义

```text
          Y+ (前方 500cm)
           ↑
     ┌─────┴─────┐
     │           │
 X-  │   宿舍    │  X+
←─── │  内部     │ ───→
165cm│           │165cm
     │           │
     └─────┬─────┘
           │
          Y=0 (雷达正下方/门口近端)
```

- **X 方向**：宽度方向，X±165cm 覆盖宿舍全宽
- **Y 方向**：长度方向，Y=0 是雷达正下方（门口近端），Y+=500cm 是宿舍远端
- **|X| < 75cm**：中间过道
- **75cm ≤ |X| ≤ 165cm**：左右两侧床/桌区
- **Y < 300cm**：下方近端区域（床下/桌前）
- **Y ≥ 300cm**：远端床面区域

## 5. 串口协议速查

### 5.1 通用帧格式

```text
53 59 | Control | Command | Len_H Len_L | Data... | Sum | 54 43
帧头     控制字     命令字      数据长度(大端)   数据     校验    帧尾
```

- **校验**：从帧头到数据区求和，取低 8 位
- **帧头**：固定 `53 59`
- **帧尾**：固定 `54 43`

### 5.2 当前脚本解析的控制字

| 控制字 | 命令字 | 含义 | 数据格式 |
|---|---|---|---|
| `0x80` | `0x01`/`0x81` | 人体存在 | 1B：1=有人，0=无人 |
| `0x80` | `0x02`/`0x82` | 运动状态 | 1B：0=无，1=静止，2=活跃 |
| `0x80` | `0x03`/`0x83` | 体动参数 | 1B：0-100 |
| `0x82` | `0x02`/`0x82` | **轨迹目标（主数据）** | N×11B |
| `0x86` | `0x0A` | 实时人数（仅上报） | ≥2B：[min,max] |
| `0x86` | `0x0C` | 精准人数（仅上报） | ≥2B：[min,max] |
| `0x01` | `0x01` | 心跳 | 记录即可 |

### 5.3 轨迹目标布局（关键：每目标 11 字节）

```text
id(1) | size(1) | feature(1) | x(2) | y(2) | height(2) | speed(2)
```

- 数据区长度 = 目标数 × 11
- `data_len == 0` 表示当前无目标
- **x/y/speed 使用符号位格式**（最高位表符号），不是补码
- **height 使用无符号 uint16**

### 5.4 符号位解析函数

```lua
local function parse_radar_signed16_be(hi,lo)
    local val=hi*256+lo
    if val>=32768 then return -(val-32768) end
    return val
end
```

示例：`80 08` → `val=32776` → `-(32776-32768)` = `-8cm`

### 5.5 主动轨迹帧示例

```text
53 59 82 02 00 0B 00 01 02 80 08 00 2E 00 B8 00 06 B2 54 43
```

解析结果：

| 字段 | 值 |
|---|---:|
| 目标 ID | 0 |
| 目标大小 | 1 |
| 目标特征 | 2 |
| X 坐标 | -8cm |
| Y 坐标 | 46cm |
| 高度 | 184cm |
| 速度 | 6cm/s |

## 6. 人数统计核心逻辑

### 6.1 人数统计口径（关键原则）

```text
people_count = confirmed track 数量（active + holding 状态）
```

| 数据来源 | 是否参与 people_count | 用途 |
|---|---|---|
| `0x82` 轨迹 confirmed track | ✅ **唯一来源** | 计算 people_count |
| `0x86/0x0A` 实时人数 | ❌ 不参与 | 仅上报，用于对照排查 |
| `0x86/0x0C` 精准人数 | ❌ 不参与 | 仅上报，用于对照排查 |
| 原始 `target_count` | ❌ 不参与 | 仅上报，用于排查 |

### 6.2 航迹状态机

```text
candidate（候选）
   │ 命中帧数 ≥ CONFIRM_HITS(4)
   │ 且累计位移 ≥ CONFIRM_MOVE_CM(60)
   │ 且运动命中 ≥ CONFIRM_MOVING_HITS(2)
   ↓
active（活跃）
   │ 丢帧 ≥ ACTIVE_TO_HOLD_SEC(2)
   ↓
holding（保持）
   │ 门口带丢失 ≥ EXIT_GRACE_SEC(3)  → remove_track（减人）
   │ 下方区域运动后丢失 ≥ LOWER_MOVING_LOST_EXIT_SEC(8) → remove_track（减人）
   │ 全屋无人 ≥ FALLBACK_ZERO_SECONDS(15) → clear_all（清零）
   │ 雷达无帧 ≥ RADAR_STALE_SECONDS(30) → clear_all（清零）
```

### 6.3 航迹确认条件

一条航迹被确认为"有人"需同时满足：

1. 按 11 字节格式解析成功
2. 高度在 `20cm ~ 245cm` 范围内
3. 命中帧数 ≥ `CONFIRM_HITS`（4 帧）
4. 从出生点到当前点最大净位移 ≥ `CONFIRM_MOVE_CM`（60cm）
5. 运动命中帧数 ≥ `CONFIRM_MOVING_HITS`（2 帧）

### 6.4 减人路径（只有以下三类减人）

| 路径 | 条件 | 参数 |
|---|---|---|
| 门口带离开 | 确认航迹最后坐标在 `EXIT_ZONE` 内，丢失超时 | `EXIT_GRACE_SEC=3` |
| 下方区域运动后丢失 | 下方区域航迹刚运动过，随后丢失超时 | `LOWER_MOVING_LOST_EXIT_SEC=8` |
| 全屋无人兜底 | `human_present=0` + 实时人数=0 + 精准人数不阻塞 | `FALLBACK_ZERO_SECONDS=15` |

### 6.5 防分裂机制

- 下方区域（Y<300cm）一人多目标拆分时，`LOWER_MATCH_DIST_CM=110` / `LOWER_SPLIT_SUPPRESS_CM=110` 优先合并近距离目标
- 不硬限制下方人数，真实多人时由彼此分离且各自满足完整轨迹条件的航迹计入

### 7.2 上报节奏

| 条件 | 行为 |
|---|---|
| 开机 | 延时 2 秒后发送 `dreg` |
| 首次上报 | `force_initial_dup=1`，不等 `radar_ready`，先推一帧覆盖云端旧值 |
| 雷达未就绪 | 不立即 `dup`，除非开机超过 30 秒 |
| 状态变化 | 触发 `dup`（任意上报字段变化即触发） |
| 无交互超时 | 180 秒触发一次心跳 `dup` |
| 主循环 | 每 100ms 轮询 |
| 云端确认 | 收到 `dupbck` 后刷新 `lasttime` |

### 7.3 状态变化检测字段

以下任一字段变化即触发 `dup`：

`human_present`, `motion`, `body_move`, `has_person`, `people_count`, `target_count`, `realtime_people_count`, `accurate_people_count`, `primary_target_id`, `traj_x`, `traj_y`, `traj_height`, `traj_speed`, `uart_bytes`

## 9. 现场测试验证清单

### 9.1 基础通信验证

- [ ] DTU 启动后 `radar_ready=1`
- [ ] `radar_frame_count` 持续增长
- [ ] `uart_bytes` 持续刷新为最新 UART 原始 HEX

### 9.2 空场验证

- [ ] 无人时 `people_count=0`
- [ ] `target_count`、`realtime_people_count`、`accurate_people_count` 是否误报（记录但不影响 people_count）

### 9.3 单人验证

- [ ] 1 人进入床下/下方区域，走完完整轨迹后 `people_count=1`
- [ ] 人静止后轨迹丢帧，`people_count` 保持 1
- [ ] 主目标坐标保持最后一次宿舍区域内坐标

### 9.4 减人验证

- [ ] 人从门口离开：门口带丢失超过 `EXIT_GRACE_SEC`(3s) 后 `people_count` 下降
- [ ] 人从下方区域离开：运动后丢失超过 `LOWER_MOVING_LOST_EXIT_SEC`(8s) 后 `people_count` 下降
- [ ] 全屋无人：超过 `FALLBACK_ZERO_SECONDS`(15s) 后 `people_count=0`

### 9.5 断电/断串口验证

- [ ] 拔掉雷达串口：超过 `RADAR_STALE_SECONDS`(30s) 后 `people_count=0` 并上报
- [ ] DTU 重新上电：首包 `dup` 覆盖云端旧人数

### 9.6 多人验证

- [ ] 多人进入下方区域，只有彼此分离且各自满足完整轨迹条件时 `people_count` 才增加
- [ ] `target_count` 可大于 `people_count`（一人多目标拆分时）
