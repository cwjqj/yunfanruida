# 云帆瑞达毫米波雷达：可配置参数与功能清单（产品经理版）

> 梳理日期：2026-07-15  
> 适用模组：云帆瑞达 R60BMP1 60GHz  
> 说明：本文区分“已经具备的功能”“现场可配置参数”和“需要研发改代码的算法参数”。当前工程同时保留侧装、顶装及多个实验版本，不能把所有能力视为已在同一固件中同时启用。

## 1. 产品能力总览

| 功能 | 当前工程状态 | 产品含义 | 主要输出 |
|---|---|---|---|
| 有人/无人检测 | 已实现、已验证 | 判断区域内是否有人 | `human_present`、`has_person` |
| 运动状态 | 已实现、已验证 | 无运动/静止/活跃 | `motion=0/1/2` |
| 体动强度 | 已实现、已验证 | 0–100，数值越大活动越强 | `body_move` |
| 区域人数统计 | 侧装、顶装均已实现 | 当前主算法按“软件确认轨迹”计数 | `people_count` |
| 雷达原始实时/精准人数 | 已接入，主要作诊断 | 模组自带人数结果，可用于校准或回退 | `realtime_people_*`、`accurate_people_*` |
| 多目标原始轨迹 | 已实现 | 输出目标 ID、位置、高度、速度 | `target0`～`target2` 等 |
| 稳定轨迹与轨迹生命周期 | 顶装 Track 2.0 可选版 | 抵抗目标 ID 跳变，输出路径、方向、active/holding 等状态 | `track0`～`track2`、`track_event` |

| OTA 升级 | 协议层支持，工程无完整安全升级工具 | 模组固件开始、分包、结束升级 | 高风险维护能力，不建议当前直接开放 |

## 3. 第一类：适合做成产品配置项的参数

这些参数有明确业务含义，适合放入出厂配置、安装工具或运维后台。修改后仍必须执行“下发—查询—读回校验”。

### 3.1 安装与空间配置

| 产品配置项 | 可选值/单位 | 侧装当前值 | 顶装当前值 | 作用与注意事项 |
|---|---|---:|---:|---|
| 安装方式 | 侧装 / 顶装 | 侧装 | 顶装 | 决定角度、高度和后续算法 profile；不能只改名称不改模组参数 |
| X/Y/Z 安装角度 | 度；协议为角度×100 | `0/0/30°` | `0/0/90°` | 协议文档范围为 -180°～180°；本项目只验证 30°侧装和 90°顶装 |
| 安装高度 | cm，步长 1cm | `220cm` | `330cm` | 必须写实际离地高度；协议字段可到 65535cm，但不代表有效物理安装范围 |
| 探测范围模式 | 自动 / 固定 | 自动范围限制 | 自动范围限制 | 两种安装方式都先写入各自房间边界，再启用自动探测和范围限制；固定模式仅作为可选回退 |
| X 正/负边界 | cm | `+165/-165` | `+165/-165` 外边界 | 定义房间横向覆盖范围 |
| Y 正/负边界 | cm | `+500/-0` | `+250/-250` 外边界 | 定义房间纵向覆盖范围 |
| 软件人数 ROI | X/Y 四个边界，cm | `X[-165,165]、Y[0,500]` | `X[-165,165]、Y[-250,250]` | 最终人数只统计软件 ROI 内确认轨迹；应与房间尺寸一致 |
| 门区开关与边界 | 开/关 + X/Y 四边界 | 主脚本内有门区 | 顶装主版默认关闭 | 进入门区后丢失可更快减人；坐标未标定时必须关闭 |
| 特殊下方/近端区域 | 开/关 + Y 边界 | 侧装 `Y<300cm` | 顶装默认关闭 | 用于侧装床下/近端目标的宽匹配与快速退出策略 |

| 配置项 | 可选值/单位 | 侧装当前值 | 顶装当前值 | 产品影响 |
|---|---|---:|---:|---|
| 人体存在功能 | 开/关 | 开 | 开 | 关闭后无法提供有人/无人业务能力，正常产品应保持开启 |
| 轨迹跟踪功能 | 开/关 | 开 | 开 | 关闭后当前软件人数、轨迹、姿态功能均无法正常工作 |

| 精准人数上报间隔 | 秒 | `60s` | `60s` | 用于慢速校准/诊断；不应描述为 1 秒实时能力 |

| 自动探测范围限制 | 开/关 | 开启 | 开启 | 两种安装方式的默认配置均开启；切换固定模式时关闭 |
| 清除人数信息 | 一次性命令 | 按需 | 按需 | 清空模组内部人数状态，适合重新标定或异常恢复 |

## 4. 第二类：可调，但应由研发/算法人员修改的参数

这些参数直接影响人数误增、漏人、静止保持和姿态误判，不建议直接暴露给普通用户。

### 4.1 人数与轨迹算法参数（当前主要默认值）

| 参数组 | 当前值 | 控制效果 | 调大/调小的典型影响 |
|---|---:|---|---|
| 有效目标高度 | `20–245cm` | 过滤地面杂波和异常高点 | 范围过窄会漏检儿童/躺卧/特殊姿态，过宽会引入杂波 |
| 内部航迹表容量 | `32` | 保存 candidate/active/holding 航迹 | 这是软件表容量，不是硬件同时探测人数承诺 |
| 同 ID 最大跳变 | `150cm` | 判断同一雷达 ID 是否仍可能为同一人 | 太大易串人，太小易断轨 |
| 普通匹配距离 | `90cm` | 跨帧轨迹匹配 | 太大易合并，太小易产生新轨迹 |
| 近端匹配距离 | `110cm` | 侧装近端/下方区域放宽匹配 | 只适合特定安装几何关系 |
| 分裂抑制距离 | 普通 `45cm`，近端 `110cm` | 抑制同一人被拆成多轨迹 | 太大会把相邻两人合并，太小会一人多计 |
| 确认命中次数 | `4` | 候选轨迹成为正式人数前需连续命中 | 越大越稳但人数出现更慢 |
| 确认累计位移 | `60cm` | 要求有真实移动证据 | 静止进入者可能更晚确认；太小则杂波更易成“人” |
| 确认运动命中 | `2` 次 | 需要多次运动证据 | 越大越保守 |
| 运动速度阈值 | `10cm/s` | 判定明显运动 | 越高越容易把慢动当静止 |
| 单步运动阈值 | `15cm` | 识别跨帧移动 | 影响候选轨迹确认速度 |
| active 转 holding | `2s` | 短暂丢失后仍保留人数 | 越长切换越慢，越短更快进入静止保持 |
| 门区退出宽限 | `3s` | 门区目标消失多久后减人 | 越短退出更快但误减风险更高 |
| 侧装近端运动后丢失 | `8s` | 特殊区域快速减人 | 需结合床下/桌前实测 |
| 候选轨迹丢失 TTL | `2s` | 删除未确认的短暂杂波 | 越长候选杂波保留越久 |
| 侧装全屋无人兜底清零 | `15s` | 多路无人证据持续成立后清空残留人数 | 太短可能误清，太长会延长幽灵人数 |
| 侧装精准人数新鲜期 | `90s` | 判断 60 秒周期的精准人数是否仍可参考 | 修改精准上报周期时必须同步复核此值 |
| 雷达断流清零 | `30s` | 无有效帧后清空旧状态 | 越短故障反馈快，但通信抖动时更易清零 |
| 弱证据全清 | 顶装 `20s` | 长时间无有效证据时清除弱轨迹 | 影响幽灵人数残留 |
| 成熟轨迹条件 | `12` 次命中且生命周期 `30s` | 区分可靠长期轨迹 | 越严格越不易进入长保持 |
| 成熟轨迹保持 | 顶装 `300s` | 静止丢帧时长期保人 | 提高静止连续性，也可能让幽灵人数残留更久 |
| 边缘丢失退出 | 顶装 `8s` | 边缘目标消失后更快移除 | 必须结合 ROI 边缘实测 |
| 轨迹稳定判定 | 相同签名 `5` 帧 | 识别重复稳定轨迹 | 影响去重和诊断字段 |
| 稳定轨迹保活上报 | `30s` | 轨迹不变时仍周期上报 | 越短云数据量越大 |
| 人数主来源 | 默认 `track`；可改 `accurate` 回退 | 决定最终 `people_count` 来源 | 切换会改变产品人数定义，必须作为版本级变更管理 |

### 4.3 顶装姿态识别参数（Posture 3.1）

| 参数 | 当前值 | 产品含义/风险 |
|---|---:|---|
| 姿态有效高度范围 | `35–230cm` | 超出范围不参与姿态判断 |
| 高度平滑窗口 | 最近 `5` 个样本中位数 | 抑制单帧抖动 |
| 初始姿态确认 | `5` 次命中且至少 `2s` | 首次输出姿态的速度与稳定性 |
| 姿态切换确认 | 基础配置 `6` 次且至少 `3s`；实际按转换类型覆盖为更严格的 `8次/2s`、`10次/3s` 或 `12次/5s` | 防止站/坐/躺频繁跳变；产品侧不宜只展示一个统一确认时间 |
| 姿态丢失保持 | `15s` | 短暂丢失时保留最后姿态 |
| 站立阈值 | 进入 ≥`145cm`；保持 ≥`125cm` | 需用不同身高人员实测标定 |
| 坐姿阈值 | 进入 `70–145cm`；保持 `65–150cm` | 站坐边界存在迟滞 |
| 躺姿阈值 | 进入 ≤`70cm`；保持 <`90cm` | 顶装测得的是目标高度特征，不应直接等同人体真实身高 |
| 过道边界 | `\|X\|<75cm` | 中间过道 |
| 床桌区边界 | `75≤\|X\|≤165cm` | 左右两侧上床下桌区 |
| 上铺床面高度 | 当前 `0cm`，表示未标定 | 未填真实床面高度前会启用保守模糊保护，床上躺姿不能作为正式能力承诺 |
| 床上躺姿相对高度 | 床面上 `15–70cm` | 需按上铺床面和不同人体姿态标定 |

|---|---:|---|

## 5. 可选择的产品方案组合

### 方案 A：基础存在感知

- 有人/无人、运动状态、体动强度；
- 适合只关心房间是否有人、活动程度的场景；
- 参数最少、风险最低。

### 方案 B：宿舍区域人数统计（当前主推）

- 基础存在感知 + 软件 ROI 人数 + 原始/精准人数诊断；
- 可选侧装 30°或顶装 90°；
- 顶装对房间覆盖更对称，侧装更依赖床、桌、过道的现场坐标标定。

### 方案 C：三人完整轨迹

- 在方案 B 上增加最多三人的稳定 UID、位置、速度、路径、方向和生命周期；
- 软件可解析最多 23 个单帧目标、内部表容量 32，但云端完整输出固定为 3 条；
- 硬件真实同时探测上限尚无充分真机结论，产品规格只能写“当前设计面向 3 人完整轨迹”。

### 方案 D：人数 + 三人姿态 + 床桌分区（实验）

- 增加站/坐/躺、姿态置信度、过道/床桌分区；
- 当前上铺床面高度仍未标定，且需采集不同身高、站坐躺样本；
- 建议先作为 PoC/试点功能，不直接进入量产承诺。

### 方案 E：门区退出增强（实验）

- 在人数统计上增加软件门区，轨迹进入门区后消失可更快确认退出；
- 需要现场采样门口坐标、连续进出、门区停留再返回等反例；
- 未完成标定前保持关闭。

## 6. 建议产品后台开放的配置层级

| 层级 | 建议开放对象 | 参数 |
|---|---|---|
| 普通用户 | 不建议开放复杂参数 | 只展示功能结果；最多提供“基础/人数/轨迹”工作方案选择 |
| 安装人员 | 安装向导 | 安装方式、实际高度、房间尺寸/ROI、自动或固定范围、门区/床区标定、串口选择 |
| 运维人员 | 高级设置 | 上报间隔、标签、干扰边、雷达功能开关、清除人数、参数读回、恢复默认 |
| 研发/算法 | 工程调参页或版本配置 | 轨迹匹配、确认、保持、退出、姿态阈值、人数来源、云端限频等全部内部参数 |

建议每个可写参数同时具备：当前值、目标值、写入结果、读回值、最后修改时间、恢复场景默认值。安装方式、范围、标签和算法 profile 应作为一个“场景配置包”原子切换，避免只改其中一项造成坐标系不一致。

## 7. 当前不能直接对外承诺的事项

1. **不是所有参数都能从云端远程修改。** 当前人数主脚本主要支持数据查询和上报；其中侧装主脚本收到 `sset` 会回复成功，但实际上没有读取或写入任何业务参数，产品端必须暂时禁用该入口或先修正回执逻辑。真正的 `sset` 安装模式写入只在独立任务/模块中实现，尚需合并到主业务任务。
2. **多个脚本代表互斥的部署版本。** 人数、Track 2.0、Posture 3.1、门区退出不是默认在一个任务中全部同时运行。
3. **23 个目标是软件解析理论上限，不是雷达硬件人数规格。** 现有资料仅记录真机日志观察到约 8 个原始目标，仍需专门极限测试。
4. **姿态识别仍需现场标定。** 尤其不同身高、床面高度、床上坐/躺和遮挡场景尚未形成量产阈值。
5. **固定范围、标签与轨迹帧存在联动风险。** 工程注释明确提示部分 `0x07` 范围/标签设置可能抑制轨迹帧，任何修改都要回归 `0x82/0x02` 是否持续上报。
6. **云端实时性受限频策略影响。** 雷达可以 1 秒上报人数，但 DTU 为控制流量对动态数据有变化触发和限频逻辑，产品页面刷新频率需与实际上云策略一致。

## 8. 产品经理建议优先确认的决策

1. 首发产品主安装方式：顶装、侧装，还是两者均支持。
2. 首发功能边界：只做存在+人数，还是同时承诺三人完整轨迹。
3. 姿态识别是否进入首发；若进入，需要安排床面高度、不同身高和三人遮挡数据采集。
4. 参数配置入口：仅 PC 安装工具，还是必须支持云端远程配置。
5. 多房型支持方式：固定宿舍模板，还是允许安装人员录入房间尺寸并生成 ROI。
6. 人数结果 SLA：出现速度、离开清零时间、静止保持时间、最大可承诺人数和误差口径。
7. 云端数据合同：哪些字段是正式业务字段，哪些仅供研发诊断，避免前台直接展示原始不稳定数据。

## 9. 硬件与安装规格约束

| 项目 | 工程资料要求 | 产品设计含义 |
|---|---|---|
| 供电 | `4.9–6V`，额定电流不低于 `200mA`，纹波不高于 `100mV` | 电源不稳会直接表现为断流、复位或结果异常 |
| 串口电平 | `3.3V TTL`，`115200 8N1` | 不是 RS-232 电平，接错可能损坏设备 |
| 天线与外壳 | 天线面距非金属平直外壳 `2–5mm` | 金属遮挡、外壳曲面或间距不合适会改变探测效果 |
| PCBA 布局 | 雷达贴件高度比周边器件至少高 `1mm` | 避免周边结构遮挡天线视场 |
| 静电防护 | 避免直接触摸天线与连接器管脚 | 安装、调试和售后流程需包含 ESD 要求 |
